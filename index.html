Closing the iOS Gap

A comprehensive strategy to fix attribution blind spots for Kick It California & ASQ.

Feb 2026 Presenter: One-Man Shop

The "Black Box" Problem

The Situation: We are fully tracked on Android (via Device ID) but flying blind on iOS. Apple's privacy changes (iOS 14.5+) block us from seeing which ads or emails drive iPhone installs.

Android Status

100% Visibility. We know exactly where users come from. We can optimize spend and retarget effectively.

iOS Status

0% Visibility. All marketing installs appear as "Organic." We cannot justify ad spend or track email conversion.

Detailed Options Analysis

We have three viable paths. Here is the brutally honest breakdown of costs, pros, and cons for our specific team size.

Option A: The Gold Standard

Branch.io

~$200/mo
Best-in-Class UX: Seamless "Deep Linking" from email to specific app screens.
Solved: Fixes iOS tracking immediately.
The "Cliff": Includes 10k Monthly Active Users. If we grow past this, prices spike significantly.
Cost: $2,400/year recurring cost.
Option B: The Smart Start

AppsFlyer

Free (Zero Plan)
Budget Friendly: Free for the first 12,000 non-organic installs (ads/emails).
Analytics Power: Industry leader in fraud detection and data accuracy.
Solved: Fixes iOS tracking immediately.
UI Complexity: The dashboard is data-heavy; slightly steeper learning curve than Branch.
Option C: The Manual Grind

HubSpot Manual

$0 (+ Labor)
No New Tech: Uses our existing HubSpot Landing Pages.
Blind Spots: We only track "Clicks," not "Installs." Low accuracy.
Labor: Requires monthly SQL/CSV exports and manual data merging.
Bad UX: Forces user to a landing page first, rather than direct-to-store.

Implementation Plan

Select a strategy to view the specific checklist and code requirements.

Branch.io Implementation Checklist

Phase 1: Dashboard Config

  • Set up Link Domain (e.g., kickit.app.link).
  • Retrieve "Live Key" and "Test Key".

Phase 2: iOS Code (The Hard Part)

  • CocoaPods: Add pod 'Branch' to Podfile.
  • Info.plist: Add keys:
<key>branch_key</key> <string>key_live_kaFuWw8...</string> <key>branch_universal_link_domains</key> <array> <string>kickit.app.link</string> </array>
  • Capabilities: Enable "Associated Domains" in Xcode and add applinks:kickit.app.link.

AppsFlyer Implementation Checklist

Phase 1: Zero Plan Setup

  • Create account and select "Zero" Plan.
  • Get Dev Key from Dashboard settings.

Phase 2: iOS Code

  • CocoaPods: Add pod 'AppsFlyerLib'.
  • AppDelegate: Initialize the SDK.
import AppsFlyerLib func application(_ application: UIApplication, didFinishLaunchingWithOptions...) { AppsFlyerLib.shared().appsFlyerDevKey = "YOUR_DEV_KEY" AppsFlyerLib.shared().appleAppID = "123456789" }
  • ATT Prompt: You MUST add the "App Tracking Transparency" prompt logic or Apple will reject the build.

Manual Implementation Checklist

Phase 1: Marketing Assets

  • Create HubSpot Landing Page (The "Gateway").
  • Create two CTA Buttons: "Download iOS" and "Download Android".

Phase 2: The Monthly Ritual

  • Export: Run SQL query on App Database for new signups (Last 30 Days).
  • Clean: Format CSV to match HubSpot import properties.
  • Import: Upload to HubSpot and map Email -> Email.
  • Update: Bulk update property App_Status to "Installed".
